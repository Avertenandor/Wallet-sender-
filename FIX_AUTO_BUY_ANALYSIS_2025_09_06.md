# Анализ и исправление автопокупки

**Дата:** 6 сентября 2025  
**Проблема:** Провал swap транзакций в автопокупке  
**Статус:** 🔧 В ПРОЦЕССЕ ИСПРАВЛЕНИЯ

## 🔍 Анализ логов

### Проблемы из логов:
1. **Approve проходит успешно** ✅
   - Gas used: 29094
   - Транзакция подтверждается

2. **Swap транзакция проваливается** ❌
   - Receipt status: 0
   - Транзакция отклонена контрактом

### Возможные причины:
1. **Недостаточный лимит газа** - установлен 100,000
2. **Проблема с проскальзыванием** - amountOutMin = 0
3. **Изменение курса** между проверкой и выполнением
4. **Неправильный путь обмена**

## 📊 Сравнение с проектом "Частокол"

### Что работает в "Частокол":
1. **Правильный slippage**: 1% (0.01) вместо 0
2. **Оценка газа**: `estimate_gas()` + 20% запас
3. **Проверка ликвидности**: перед каждой транзакцией
4. **Retry механизм**: для всех операций
5. **Валидация параметров**: перед выполнением
6. **Правильный путь**: USDT -> WBNB -> Token

### Что не работает в WalletSender:
1. **amountOutMin = 0** - нет защиты от проскальзывания
2. **Фиксированный газ** - не используется estimate_gas()
3. **Неправильный путь** - USDT -> PLEX ONE (прямой)
4. **Нет retry** для критических операций

## 🔧 План исправлений

### 1. Исправить slippage protection
```python
# Было:
amountOutMin = 0

# Стало:
slippage = 0.01  # 1%
amountOutMin = int(expected_amount * (1 - slippage))
```

### 2. Добавить оценку газа
```python
# Добавить:
estimated_gas = self.web3.eth.estimate_gas(transaction)
transaction['gas'] = int(estimated_gas * 1.2)  # +20% запас
```

### 3. Исправить путь обмена
```python
# Для USDT покупок использовать:
path = [USDT_ADDRESS, WBNB_ADDRESS, token_address]
# Вместо прямого пути USDT -> PLEX ONE
```
